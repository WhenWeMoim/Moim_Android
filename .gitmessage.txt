
#
#
#######   Title 작성 방법   #########
#
# "[타입] 제목" 형식으로 간단히 작성
# 타입은 맨 아래 목록 참고
# 제목은 변경사항을 짧고 명확하게 영어로 표기(제목 끝에 마침표 금지)
#
#######   Body 작성 방법   #########
#
# 본문(추가 설명)을 작성
# '어떻게'가 아닌 '무엇을 왜' 했는지를 중심으로 서술
#
#######   Tag 작성 방법    #########
#
# 꼬릿말 작성. 해결중인 내용이나, 어려운 부분 등의 상세 설명 추가
#
#######   TItle타입 목록    #########
|타입|타입 설명|
|---|---|
|feat|새로운 기능 추가|
|fix|버그 수정|
|modify|코드 수정|
|remove|문서, 내용 삭제|
|test|테스트 코드 추가|
|annotation|주석 추가|
|style|코드 의미에 영향을 주지 않는 변경사항|
|chore|빌드 부분 혹은 패키지 매니저 수정사항|
